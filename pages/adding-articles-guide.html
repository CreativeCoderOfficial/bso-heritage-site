1 · When should I write an article?

Every item in the collection already has a metadata record — the structured information in the Google Sheet (name, dates, country, district, and so on). That is always enough for browsing and searching.

If you want to know more about the Google sheet and how to add simple items in general, check the main overview guide.

An article is for when you want to tell a proper story: paragraphs of history, photographs, lists of notable members, timelines, and quotes. Think of the metadata as a museum label and the article as the full exhibition panel.

Good candidate for an articleProbably doesn't need one
A group with 50 years of history and notable membersA badge whose description fits in two sentences
A Far & Wide issue covering a major world jamboreeA document that speaks for itself
A group that has closed and whose story would otherwise be lostA newly registered group with no history yet
Articles are optional. An item works perfectly on the site without one — you can always add an article later.

3 · Your first time on GitHub

GitHub is where the website's files live. You'll use it entirely through a web browser — no software to install. Here is what the articles folder looks like.

Never used GitHub before? You only need three actions: (1) click a folder to open it, (2) click "Add file" → "Create new file" to create a new file, (3) scroll to the bottom and click the green "Commit changes" button to save. That's it.

4 · Creating subfolders to stay organised

If you have many related articles — for example, all articles about groups in Germany, or all articles from a particular era — you can organise them into subfolders. This keeps the articles folder tidy as the collection grows.

GitHub cannot create an empty folder. Every folder must contain at least one file. The trick is to create the folder and a file at the same time, by typing the folder name and a slash into the filename box.

How to create a subfolder

1

Go to the articles folder and click "Add file" → "Create new file"

You'll see a filename input box at the top of the editor.

2

Type the folder name, then a slash, then the filename

As soon as you type a /, GitHub turns what you typed before it into a folder. For example, typing:

germany/grp-de-001.md

...creates a folder called germany and puts your file inside it. You'll see the folder appear visually as you type.

3

Update the permalink in the front matter to match

If your file is in a subfolder, the permalink in the front matter must reflect that:

permalink: /articles/germany/grp-de-001.html

And when you add the link in the Google Sheet, use the same path:

/articles/germany/grp-de-001.html
If you want to create a folder that will hold many files later but you're not ready to add them yet, create a placeholder file called placeholder inside it instead of an actual filename. This is just an empty file whose only purpose is to keep the folder from disappearing. You can delete it later once real article files are in the folder.

5 · Step-by-step: create your article file

1

Find the item's objectid in the Google Sheet

Open the BSO Heritage Google Sheet. Find the row for the item you want to write about. Copy the value in the objectid column exactly — it looks something like grp-be-001.

The objectid is case-sensitive. Copy it exactly — grp-be-001 and GRP-BE-001 are treated as different things.
2

Navigate to the articles folder in GitHub

In the repository, click through: pages → articles. You'll see the list of existing article files.

3

Open the template and copy its contents

Click on _template.md to open it. Click the pencil icon ✏️ in the top-right corner of the file. Select all the text (Ctrl+A on Windows, Cmd+A on Mac) and copy it (Ctrl+C / Cmd+C).

4

Create a new file

Go back to the pages/articles/ folder. Click "Add file" → "Create new file". In the filename box, type your filename using the objectid: grp-be-001.md. Then paste the template content into the editor below.

5

Fill in the front matter and write your article

Replace the placeholder values in the top section (the front matter — see Section 6), then write your article below the second --- line (see Section 7 for formatting help).

6

Save the file by committing it

Scroll to the bottom of the page. You'll see a "Commit changes" panel. Optionally write a short note like "Add article for 1st Brussels Group". Click the green "Commit changes" button.

6 · Understanding the front matter

The front matter is the block at the very top of every article file, surrounded by triple dashes (---). It tells the website what kind of page this is. Do not delete the dashes.

📄 Front matter — fill in the highlighted fields
---
layout: article             # Leave this exactly as is. Never change it.
objectid: grp-be-001        # Copy EXACTLY from the Google Sheet objectid column.
title: 1st Brussels Group   # The display name shown on the page.
type: Group                 # Must be one of: Group, Badge, Document, Far & Wide
permalink: /articles/grp-be-001.html  # Replace with your objectid. Must start with /articles/
---
FieldWhat to enterCommon mistake
layout Always article — never change this Deleting or changing it breaks the page entirely
objectid Copy character-for-character from the Google Sheet's objectid column Typing it by hand and getting a letter wrong — the metadata panel won't load
title The name of the group / badge / document as you want it displayed Leaving it as REPLACE-WITH-TITLE
type Exactly one of: Group, Badge, Document, Far & Wide Using lowercase (group) — the navigation trail back-link breaks
permalink /articles/ + your objectid + .html Forgetting the leading / or the .html at the end
The type field is case-sensitive. Group works. group, GROUP, or groups do not. The same applies to Badge, Document, and Far & Wide.

8 · Adding images

You can embed photographs and scanned documents in your article. The template already includes an example — all you need to do is replace the two parts shown in bold below.

In the template you'll find this line:

![Image description]({{ '/objects/your-image.jpg' | relative_url }})

Replace Image description with a short description of what the photo shows (this helps visually impaired readers). Replace your-image.jpg with the actual filename of your image. Leave everything else exactly as it is.

Before you can reference an image, you need to upload it

Go to the objects/ folder in the repository. Click "Add file" → "Upload files". Drag your image in and click "Commit changes".

Image filename rules — use only lowercase letters, numbers, and hyphens. No spaces.
✅ grp-be-001-camp-1962.jpg
🚫 Group photo (Brussels) 1962.jpg
Keep image files under 2MB. Larger files slow the page down. You can resize images for free at squoosh.app before uploading.

9 · The workflow: what order to do things

Here is the recommended order for creating a new article from start to finish:

1. Write your .md file in GitHub
→
2. Commit the file
→
3. Update article_url in the Sheet
→
4. Changes appear within minutes

You can also update the Google Sheet first if you prefer — the order is flexible. What matters is that both steps are done before you expect the article link to work: the file must exist in GitHub, and the article_url must be filled in the Sheet. If only one of the two is done, visitors won't see the article link yet, but nothing will break.

Images can be uploaded at any time — before or after committing the article file. If an image is missing when a visitor first loads the page, it simply won't appear. Once you upload it and the site rebuilds, it will show up automatically.

10 · When will my changes appear on the site?

Every time a file is saved (committed) to GitHub, the website automatically rebuilds itself. This usually takes 2–5 minutes. After that, your changes will be live.

This means:

  • A new article file you commit will appear on the site within a few minutes
  • An image you upload to objects/ will appear within a few minutes
  • Changes you make only in the Google Sheet (without committing any file) will appear the following morning, when an overnight automatic rebuild runs
If you update the Sheet and commit a file at the same time — for example, filling in article_url in the Sheet and then saving your article file — everything updates together within a few minutes.

10 · Quick checklist

Run through this before you consider the article done:

  • I found the objectid in the Google Sheet and copied it exactly
  • My file is named [objectid].md and is in the pages/articles/ folder (or a subfolder)
  • The layout field says article and I haven't changed it
  • The objectid in the front matter matches the Google Sheet value exactly (including uppercase/lowercase)
  • The type field is exactly one of: Group, Badge, Document, Far & Wide
  • The permalink starts with /articles/ and ends with .html
  • I have committed the file to GitHub (green "Commit changes" button clicked)
  • Any images I reference have been uploaded to the objects/ folder
  • I have added the article_url in the Google Sheet, starting with /
  • In conclusion, the first five columns in the Google sheet are correctly filled in./

12 · Troubleshooting

ProblemMost likely causeFix
Metadata panel says "not found" The objectid in the front matter doesn't match the Sheet Open the Sheet, copy the objectid exactly and paste it into the front matter
Clicking the item still goes to the old page, not the article The article_url in the Sheet is missing or the Sheet hasn't been republished Check the article_url column for that item, make sure it starts with /, and republish the Sheet
Changes aren't showing up yet The site hasn't finished rebuilding Wait 5 minutes and refresh the page. Sheet-only changes may take until the next morning.
Navigation trail shows "Browse" instead of "Groups" / "Badges" etc. The type field is misspelled or has the wrong capitalisation Fix it to exactly: Group, Badge, Document, or Far & Wide
Image not showing on the article page Image not uploaded yet, or filename is wrong Check the file exists in objects/ and the filename in your markdown matches exactly
Bold or italic not working Space between the asterisks and the text, or only one asterisk instead of two Use **text** for bold (no spaces). Use *text* for italic.
Still stuck? Post in the team channel with the objectid of your item and a description of what you're seeing.
